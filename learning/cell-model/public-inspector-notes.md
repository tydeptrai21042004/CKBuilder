# Cell Model learning prompts for the Public Inspector

Complete these prompts in your own words after running the project:

1. Why is an output Cell consumed rather than modified in place?
2. Which input authorises credential creation or revocation?
3. What does the Lock Script protect in this project?
4. What invariant does the Type Script protect?
5. Why does `ACTIVE → REVOKED` require one group input and one group output?
6. Why is per-lineage irreversibility not the same as global uniqueness?
7. Why should a public verifier need RPC access but no signer?
8. What does a duplicate live Cell mean, and why should the inspector return a conflict?

Attach one transaction diagram and reference the corresponding local or public transaction evidence.
